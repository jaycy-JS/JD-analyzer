# QA Plan — JD 분석기

QA 서브 에이전트가 이 파일을 읽고 테스트를 실행합니다.  
메인 에이전트는 기능 구현만, QA 에이전트는 검증만 담당합니다.

---

## 에이전트 역할 분리

```
┌─────────────────────┐     기능 완성      ┌─────────────────────┐
│   메인 에이전트      │ ───────────────▶  │   QA 에이전트        │
│   (기능 구현)        │                   │   (검증 전용)        │
│                     │ ◀───────────────  │                     │
│  - index.html 수정  │  버그 리포트 전달  │  - qa-plan.md 읽기  │
│  - docs/*.md 업데이트│                   │  - 케이스 실행       │
└─────────────────────┘                   │  - qa-log.md 기록   │
                                          └─────────────────────┘
```

**트리거 조건**
- 새 기능 구현 완료 시 메인 에이전트가 명시적으로 요청
- 호출 방법: `@QA index.html 변경됨. TC-파일읽기, TC-분석플로우 검증해줘`

## 버그 리포트 형식

QA 에이전트가 `docs/qa-log.md`에 기록할 때 아래 형식을 사용합니다.

```markdown
## QA-NNN · {테스트명}
**날짜:** YYYY-MM-DD
**담당:** QA 에이전트
**실행 케이스:** TC-XXX-XX, TC-XXX-XX

### 버그
| ID | TC | 증상 | 심각도 |
|----|----|------|--------|
| B-NNN | TC-XXX-XX | 재현 증상 | 🔴 Critical / 🟡 Medium / 🟢 Minor |

### 통과
TC-XXX-XX, TC-XXX-XX ... N개 통과

### 메인 에이전트 전달 사항
- [ ] B-NNN 수정 요청: {설명}
```
